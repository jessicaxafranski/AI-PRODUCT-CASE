## AI Apartment Agent – Product Case Study

This repository documents the end‑to‑end product case for the **AI Apartment / Housing Opportunity Agent** — an AI‑powered assistant that helps newcomers relocating to Barcelona find apartments that meet both administrative (especially **empadronamiento**) and personal requirements.

The goal of the product is to:

- Reduce the time and complexity involved in the apartment search.
- Aggregate fragmented listings from multiple platforms and channels.
- Analyze unstructured listing text with AI to extract key signals (price, location, empadronamiento compatibility, minimum stay, trust signals, etc.).
- Surface high‑probability, trustworthy opportunities so users can contact landlords faster.

This case study is structured across classic product phases:

- **1. Problem Discovery** – Context, user behavior, and key pain points.
- **2. User Research** – Personas, needs, and frustrations.
- **3. Product Definition** – Vision, value proposition, and core features.
- **4. AI Solution Design** – How AI is used, architecture, and technical approach.
- **5. MVP Definition** – Narrowed scope, features, and success metrics.
- **6. Prototype & Agent Experiments** – Early flows, prompts, and agent behavior.
- **7. Validation & Next Steps** – How to evaluate and iterate.

Use this file as the high‑level narrative, then dive into the phase‑specific documents listed in the main `README.md`.
